These were used only as discovery signals and validated against official Microsoft documentation before inclusion.

Coverage notes:

- This task is centered on Azure Monitor, but coverage is fragmented across workspace architecture, log sources, diagnostic settings, DCRs, agents, application telemetry, identity logs, network logs, SIEM integration, and export or archival destinations.
- The most important design distinction is what gets collected, where it is routed, how long it is retained, and whether the use case requires querying, archival, streaming, or security correlation.
- Download first: Azure Monitor, Monitoring and health, Network Watcher, Microsoft Sentinel, Azure Data Explorer, Azure Event Hubs, Azure Storage, and Azure Well-Architected Framework.
- Architecture tradeoffs are more important than deep deployment steps: expect scenarios around one versus multiple workspaces, operational versus security log separation, regional placement, retention, data access, cost, and third-party SIEM integration.
- Lightly covered but testable adjacent topics include Azure Policy for enforcing diagnostic settings at scale, Azure Arc for hybrid collection, Defender for Cloud for security posture and alerts, and Container insights for AKS-specific logging.
