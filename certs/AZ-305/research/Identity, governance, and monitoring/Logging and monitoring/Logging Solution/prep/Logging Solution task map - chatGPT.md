Forum-discovery note: Public candidate discussions commonly surface Azure Monitor, Log Analytics workspaces, Application Insights, Microsoft Sentinel, diagnostic settings, VM agents, and network flow logs for AZ-305 logging and monitoring scenarios. These were used only as discovery signals and validated against official Microsoft documentation before inclusion.

Coverage notes:

* This task is centered on Azure Monitor, but the coverage is fragmented across Azure Monitor Logs, workspace architecture, data collection, agents, diagnostics, application telemetry, identity logs, Kubernetes logs, network logs, and SIEM integration.
* Architecture guidance is more useful than a single product root because AZ-305 can test tradeoffs such as one versus multiple workspaces, operational versus security log separation, regional placement, retention, data access, cost, and integration with Sentinel or third-party SIEMs.
* Download first: Azure Monitor, Monitoring and health, Network Watcher, Microsoft Sentinel, Azure Data Explorer, and Azure Well-Architected Framework.
* Major products that could be testable but are only lightly covered here: Event Hubs for streaming logs, Azure Storage for archive/compliance retention, Defender for Cloud for security posture and alerts, Azure Arc for hybrid collection, and Azure Policy for enforcing diagnostic settings at scale.
