Forum-discovery note: Public candidate discussions commonly highlight the IaaS vs. PaaS migration decision, Azure Migrate assessments, rehost vs. replatform strategy, App Service modernization, VMware migration, Azure VMware Solution, compute service selection, and case-study-style recommendation scenarios. These signals were used only for discovery and validated against official Microsoft documentation before inclusion.

Coverage notes:
- This task is fragmented across several documentation sets; Azure Migrate and the Cloud Adoption Framework migration guidance form the strongest core.
- Start with Azure Migrate, Cloud Adoption Framework migration guidance, and Azure Architecture Center compute guidance because they anchor assessment, strategy, and service-selection decisions.
- The compute decision tree and managed-services guidance are high-value for recommendation questions that compare IaaS, PaaS, containers, and serverless options.
- App Service, AKS, Container Apps, Azure Virtual Machines, and Azure VMware Solution are supporting target services that become relevant depending on workload constraints and modernization goals.
- Azure landing zones are important for target-environment readiness but should support the migration recommendation rather than dominate the map.
- Azure SQL Managed Instance and Azure Database Migration Service are adjacent to this task and more directly aligned with database migration, but they can appear in combined workload migration scenarios.
- Azure Site Recovery, Azure Arc, Azure Data Box, ExpressRoute, VPN Gateway, Azure Monitor, Azure Policy, and Cost Management support migration execution and operations but are not the core documentation sets for this specific task.
