Potentially relevant products considered: Azure Migrate, Azure Site Recovery, Azure Database Migration Service, Azure App Service, Azure Kubernetes Service (AKS), Azure Virtual Machines, Azure Virtual Network, Azure ExpressRoute, Azure VPN Gateway, Azure File Sync, Azure Data Box, Azure SQL Database (for PaaS DB target), Azure Storage (Blob/Files).

Forum-discovery note: Public discussions emphasize the importance of structured migration strategies (the 6 R’s: Rehost, Refactor, etc.), using Azure Migrate for discovery/assessment, and choosing between lifting VMs (IaaS) or using PaaS (App Service, AKS). We used these as signals but validated all items against official Microsoft docs.

Coverage notes:

- No single document covers this task fully. It spans migration tooling and service documentation across compute, data, and networking. Key documentation sets include Azure Migrate (for unified assessment/migration), Azure Database Migration Service (for databases), and service docs for target platforms (App Service, AKS, VMs, networking).
- Architecture guidance (e.g. compute decision trees, Cloud Adoption Framework) complements product docs by framing design decisions (lift-and-shift vs refactor, connectivity options).
- The highest-priority downloads would be Azure Migrate, Database Migration Service, and App Service docs, since they directly address migrating apps and data.
- Additional important references include networking docs (ExpressRoute/VPN/VNet) and Azure File Sync/Data Box for bulk data scenarios.
- Some related topics (e.g. Azure Site Recovery, Azure Functions) may appear on the exam but are covered peripherally here (ASR mainly for DR, functions for modernization). The map focuses on the core services and guidance needed to recommend IaaS vs PaaS migration approaches.
