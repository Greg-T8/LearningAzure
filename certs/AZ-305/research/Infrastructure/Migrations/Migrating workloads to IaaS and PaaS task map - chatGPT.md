These were used only as discovery signals and validated against official Microsoft documentation before inclusion. ([Reddit][2])

Coverage notes:

* This task is **fragmented across several product areas**, not covered by one documentation set alone.
* **Azure Migrate** is the primary documentation set to download first because Microsoft explicitly positions it for workload inventory, IaaS/PaaS assessment, planning, migration, and modernization. ([Microsoft Learn][3])
* **Cloud Adoption Framework migration guidance** is important because AZ-305 tests recommendation logic: migration waves, dependencies, cutover, rollback, readiness, and business constraints. ([Microsoft Learn][4])
* **Azure Architecture Center compute guidance** is more useful than a single product root when deciding between IaaS, PaaS, containers, and serverless. ([Microsoft Learn][5])
* **App Service, AKS, Container Apps, and Azure VMware Solution** are likely supporting targets depending on workload constraints: web app modernization, containerization, managed Kubernetes, serverless containers, or VMware continuity. ([Microsoft Learn][6])
* Major products that could be testable but are only lightly covered here: Azure Site Recovery, Azure Arc, Azure Data Box, ExpressRoute, VPN Gateway, Azure Monitor, Azure Policy, and Cost Management. These support migration execution and operations but are not the core documentation sets for this specific task.
