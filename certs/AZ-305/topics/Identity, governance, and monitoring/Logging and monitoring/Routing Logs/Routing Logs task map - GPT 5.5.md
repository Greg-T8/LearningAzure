Coverage notes:

- This task is anchored in Azure Monitor, but the useful documentation is fragmented across diagnostic settings, DCRs, agents, workspace design, export rules, identity logs, network logs, and destination services.
- The main design decision is which routing mechanism matches the log source: diagnostic settings for platform, activity, resource, and Microsoft Entra logs; DCRs and Azure Monitor Agent for guest OS and workload logs; Azure Monitor pipeline for centralized hybrid or multicloud ingestion; data export for post-ingestion routing.
- Destination selection is central: use Log Analytics for query, alerting, workbooks, and Sentinel integration; Storage for archive, immutable retention, and compliance; Event Hubs for streaming to external SIEM or analytics tools; Sentinel for security operations.
- Download first: Azure Monitor, Monitoring and health, Azure Event Hubs, Azure Blob Storage, Microsoft Sentinel, and Network Watcher.
- Architecture guidance is more useful than a single product root because routing questions usually combine source type, destination, retention, region, access model, cost, and compliance requirements.
- Lightly covered but testable adjacent topics include Application Insights telemetry export, Container insights DCR configuration, Defender for Cloud security data, Azure Data Explorer for high-volume analytics, Azure Native Integrations for partner observability platforms, and Azure Arc for hybrid routing.
