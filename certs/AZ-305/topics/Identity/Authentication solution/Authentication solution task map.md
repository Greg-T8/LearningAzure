These discussions were used only as discovery signals; included topics were validated against official Microsoft documentation.

Coverage notes:

- Coverage is fragmented across workforce, hybrid, application, workload, external, and legacy authentication documentation sets; no single product root covers the full task.
- Start with Azure Architecture Center and Azure Well-Architected Framework for the design lens, then use Authentication, Hybrid identity, Conditional Access, Microsoft identity platform, and Managed identities for the primary comparisons.
- Add External ID, Workload ID, Domain Services, Application Proxy, or enterprise SSO when the scenario explicitly mentions customers or partners, non-Azure workloads, legacy protocols, private applications, or federated SaaS applications.
- Modern recommendations should generally lead with phishing-resistant passwordless methods and Conditional Access authentication strengths rather than SMS or voice MFA.
- Enable password hash synchronization alongside pass-through authentication or federation when possible to improve cloud-outage resilience, enable leaked-credential detection, and support Domain Services; use federation when a requirement cannot be met natively by Microsoft Entra ID.
- Authentication strengths, Continuous Access Evaluation, Temporary Access Pass, seamless SSO, Private Access, Global Secure Access, and Verified ID are relevant adjacent topics but are covered more lightly than the core documentation sets.
