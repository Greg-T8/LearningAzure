Forum-discovery note: Public candidate discussions consistently rank identity and authentication as a heavily weighted AZ-305 area and stress requirement-driven service selection, hybrid authentication tradeoffs (PHS vs. PTA vs. federation), MFA and Conditional Access, and secretless workload/service authentication. These were used only as discovery signals and every topic was validated against official Microsoft documentation before inclusion.

Coverage notes:

- The recommendation is deliberately fragmented across documentation sets because workforce, hybrid, application, workload, external, and legacy authentication each use a different decision model; there is no single product root that covers the whole task.
- Start with Azure Architecture Center and the Well-Architected Framework for the design lens, then use the product sets for the specific comparison the scenario demands.
- Download first: Azure Architecture Center, Azure Well-Architected Framework, Authentication, Hybrid identity, Conditional Access, Microsoft identity platform, and Managed identities for Azure resources. These cover the majority of realistic scenarios.
- Add External ID, Workload ID, Domain Services, and Application Proxy when a scenario explicitly mentions customers/partners, non-Azure or external workloads, legacy Kerberos/NTLM/LDAP protocols, or private on-premises web apps.
- Currency note: Microsoft's default guidance is now phishing-resistant, passwordless authentication (Windows Hello for Business, passkeys/FIDO2, CBA) with Conditional Access authentication strengths — a modern recommendation should lead with these rather than SMS/voice MFA.
- Currency note: Microsoft Entra Private Access (part of Global Secure Access / Security Service Edge) is the strategic superset of Application Proxy, extending Zero Trust, per-app, VPN-less access with Conditional Access to any private resource, port, or protocol. Expect it to appear as the modern answer for remote access to private apps; Application Proxy remains the simpler, well-established option for pre-authenticating private web apps.
- Hybrid tradeoff to remember: enable password hash synchronization regardless of the chosen method, because it provides cloud-outage resilience, is a prerequisite for Microsoft Entra ID Protection leaked-credential detection, and supports Microsoft Entra Domain Services; federation (AD FS) is warranted only when a requirement Microsoft Entra ID cannot meet natively exists.
- Lightly covered but testable adjacent topics: authentication strengths, Continuous Access Evaluation, Temporary Access Pass, seamless SSO, and Microsoft Entra Verified ID for high-assurance account recovery.
