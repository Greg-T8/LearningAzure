Forum-discovery note: Public candidate discussions consistently point to requirement-driven choices among Azure Monitor, Log Analytics, Application Insights, VM insights, Microsoft Sentinel, Service Health, and the collection path built from Data Collection Rules and Data Collection Endpoints. These were used only as discovery signals and validated against official Microsoft documentation.

Coverage notes:

- Azure Monitor is the primary documentation set, but a complete recommendation spans application, infrastructure, Kubernetes, network, identity, security, Azure platform health, visualization, and enterprise operating-model guidance.
- The key distinction from the sibling logging and log-routing tasks is the outcome after telemetry is available: measure workload health, visualize it for the right audience, detect actionable conditions, and notify or automate the appropriate response.
- Match the tool to the requirement: Application Insights for APM and distributed tracing; VM and Kubernetes monitoring for workload-specific insights; managed Prometheus and Grafana for cloud-native metrics; Service Health for Azure-provider events; Network Watcher for IaaS network health; and Defender for Cloud or Sentinel for security requirements.
- Architecture guidance is more useful than any single product overview for this recommend-style task. The Well-Architected Framework covers workload-level monitoring-system design, while the Cloud Adoption Framework covers estate-wide scope, ownership, standardization, and hybrid or multicloud operations.
- Download first: Azure Monitor, Azure Well-Architected Framework, Cloud Adoption Framework, Azure Service Health, and Network Watcher. Add Monitoring and health, Microsoft Defender for Cloud, and Azure Managed Grafana when identity, security, or cloud-native dashboard requirements appear.
- Lightly covered but testable adjacent topics include Azure Advisor for optimization recommendations, Azure Policy and Azure Monitor Baseline Alerts for monitoring at scale, Azure Arc for hybrid resources, Azure Data Explorer for custom high-volume telemetry analytics, and Microsoft Sentinel for SIEM and SOAR.
- Azure Monitor health models remain in preview and are worth understanding conceptually for business-aware health rollup and alert-noise reduction. Azure Monitor SCOM Managed Instance was excluded because it is no longer supported and is scheduled for deprecation on September 30, 2026; System Center Operations Manager remains a conditional legacy option.
