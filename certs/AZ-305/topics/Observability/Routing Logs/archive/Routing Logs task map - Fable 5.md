Coverage notes:

- The task is anchored in a single product family — Azure Monitor — but the routing content is fragmented across several subsections of its documentation set (platform, data-collection, logs, and agents), so downloading the Azure Monitor documentation covers most of the task in one PDF.
- The exam decision has two halves: pick the routing mechanism by log source (diagnostic settings for platform/resource/activity/Entra logs; DCRs plus Azure Monitor Agent for guest OS and custom logs; Log Analytics data export for post-ingestion routing), then pick the destination by requirement (Log Analytics for analysis and alerting, Storage for cheap immutable archive, Event Hubs for external streaming, Sentinel for security operations, partner solutions for third-party platforms).
- Memorize the routing constraints that scenarios hinge on: maximum of five diagnostic settings per resource, one destination of each type per setting, same-region requirement for Storage and Event Hubs destinations, and the 90-day default activity-log retention that motivates export.
- Download first: Azure Monitor, then Monitoring and health (Microsoft Entra), Azure Event Hubs, Azure Blob Storage, and Microsoft Sentinel.
- Architecture guidance (workspace design) matters as much as any single feature page here, because routing recommendations combine source type, destination, region, ownership, retention, and cost into one answer.
- Lightly covered but testable adjacent topics: DCR transformations for ingestion cost control, Azure Policy remediation to enforce routing on existing resources, the Azure Monitor pipeline for edge and multicloud ingestion, Azure Arc for hybrid sources, and Network Watcher flow logs as a storage-first routing path.
